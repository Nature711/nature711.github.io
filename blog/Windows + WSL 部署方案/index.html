Windows + WSL 部署方案

Recommended Setup

WSL2 with Docker CLI installed + Docker Desktop for Windows on the Windows host

Why This Setup Works Best

  • WSL2: Where most Linux-based development takes place (e.g., coding, running services).

  • Docker CLI in WSL2: Lets you run Docker commands directly inside WSL for convenience.

  • Docker Desktop for Windows:

    • Provides the actual Docker daemon (dockerd), required to run containers.
    • Seamlessly manages integration with WSL2 — including networking, socket communication, volumes, and performance tuning.
    • Offers a GUI to manage containers, images, volumes, networks, and settings.

Understanding the Components

WSL2

  • A full Linux kernel running inside a lightweight VM on Windows.
  • Enables a native Linux development experience within Windows.

Docker Desktop for Windows

  • A GUI + background service that manages Docker on Windows.

Key Components

  • Docker Daemon (dockerd): The background engine that runs containers.
  • WSL2 Integration: Shares the Docker daemon with selected WSL2 distros.
  • UI/CLI Settings: Manage settings like resource limits, integrations, etc.

Common Pitfall

Problem

  • Installing Docker inside WSL2 using `sudo apt install docker
  • and assuming it works out of the box.

Reality

  • This only installs the Docker CLI in WSL2.
  • No dockerd is running, and WSL2 doesn’t support it easily without systemd.
  • Docker commands will fail unless Docker Desktop is running and providing the daemon.

How to Verify

# Check if Docker CLI is installed
which docker
docker --version

# Check if the Docker daemon is running in WSL2
ps aux | grep dockerd

Example output:

nature  5184  0.0  0.0  4028  1920 pts/8  S+ 18:12 0:00 grep --color=auto dockerd

Only the grep process is shown. This means no dockerd is actually running inside WSL2.

Docker Architecture Overview

Core Components

  • Docker Daemon (dockerd):

    • The backend engine that manages containers, images, volumes, networks.

  • Docker CLI (docker):

    • The frontend command-line tool that communicates with the daemon.

How It Works with Docker Desktop

  1. Docker Desktop starts a dockerd process on the Windows side.
  2. WSL2 integration mounts a Unix socket (Docker’s API endpoint) into your WSL2 distro.
  3. Docker CLI in WSL2 is configured (via ~/.docker/config.json) to connect to this socket.
  4. So when you run docker run in WSL2, you’re talking to the Windows-side daemon.

Even though you’re running Docker commands inside WSL2, the containers themselves are managed by Docker Desktop on Windows.

Networking Behavior

With WSL2 integration enabled:

  • WSL2 and Windows share the same loopback interface (localhost).
  • This allows seamless networking across environments.

Example

  • After running docker run -p 8080:80 nginx (i.e. maps Window hosts port 8080 to container’s port 80)

    • in WSL2: curl localhost:8080
    • in Windows host: accessing http://localhost:8080

  • Both work because:

    • The NGINX container is running on the Windows-side Docker daemon.
    • Port 8080 is published to Windows, and WSL2 shares that interface.
