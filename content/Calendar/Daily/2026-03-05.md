## Idea Inbox

### Work

今天主要还是在推进 **Food Voucher Customization** 的 TD，仍然卡在同一个问题上。很多地方理解得不是很清楚，只能不断开新的 AI Chat 去问，希望能慢慢拼出完整的理解。不过越看有时候反而越迷茫。

感觉这里可能没有什么 shortcut，最终还是得 **自己去 trace request flow**，才能真正理解系统的行为。

今天还有一个比较大的收获，是关于 **end-to-end understanding** 的反思。  
因为今天部署了第一个 Warm-up Task 的改动，上线之后需要去验证改动是否真的生效，于是开始看 log。

这时我才意识到一件有点尴尬的事情：  
我甚至没有完全搞清楚 **expected error 是什么**。

原本系统在 batch size 超过限制时会报 `Batch size exceeded`。  
而这次改动引入 batching 逻辑之后，预期行为是：

- 一个 size 为 n 的 batch
    
- 会被拆分成 `n / 50` 次 API 调用
    
- 从而避免触发这个 error
    

回头看才发现，这些其实是 **一开始就应该完全搞清楚的东西**。

为什么我直到最后验证的时候才真正理解？  
可能是因为之前只是 **局部理解（local understanding）**：知道要改哪个函数、实现什么逻辑。

但没有从 **系统层面（global understanding）** 去看：

- 这个逻辑在整个 system 里的位置
    
- request flow 是怎么走的
    
- 改动会影响哪些上下游
    

之后做任务时，也许应该先从 **整体 flow** 入手，而不是只盯着某一个局部实现。

### Content Creation / Vlog

昨天发了第一个 vlog，内容大概是：

- 上班的一天
    
- 打网球
    
- 自己研究一些 AI 工具
    

拍摄、剪辑、字幕、封面其实都花了不少心思，发布时间也特意选在 **下午六点（下班时间）**。

不过现实和预期还是有差距。  
到现在为止大概 **100 多个 views**，点赞和评论也比较少，而且大部分还是熟人。

不过做这种事情本来就需要 **脸皮厚一点**。  
更重要的是把它当成一个尝试。

虽然流量没有达到预期，但还是收获了一些有意思的反馈。

比如泽邦联系我，说他有朋友在做 **OpenClaw 的部署服务**。  
很有意思的一点是：OpenClaw 火了之后，**最先赚钱的反而是“上门安装服务”**。

听起来很传统，但其实很像一句老话：

> 当淘金热来临时，最赚钱的往往是卖铲子的人。

related article: [做淘金热里的“卖铲人”](https://zhuanlan.zhihu.com/p/1988300093158348761#:~:text=1.%20%E6%BA%90%E5%A4%B4%E9%9A%90%E5%96%BB%EF%BC%9A%E2%80%9C%E6%B7%98%E9%87%91,%E4%BE%9B%E8%B4%A7%EF%BC%8C%E7%8E%B0%E9%87%91%E6%B5%81%E5%81%A5%E5%BA%B7%E3%80%82)

技术本身是一回事，但真正的商业机会往往来自 **围绕技术的服务和商业模式**。

很多人其实并不理解这些技术，但又想尝试。  
对我来说可能只是一个 command 就能解决的事情，但对很多人来说：

- 不会命令行
- 不知道怎么配置环境

于是就变成了一个“很难”的事情。

某种程度上，这其实就是 **信息差**。

以前我可能会有点鄙视这种赚钱方式，但现在觉得其实没必要。  
短期来看，这确实是一种真实存在的商业机会。  
长期来看，信息差可能会被逐渐填平，但在某个阶段确实可以创造价值。

所以与其鄙视，不如去理解背后的 **人性和需求**。

### AI / FOMO

最近也看到不少朋友在尝试各种 AI 工具，即使他们本身没有太多技术背景。

很多时候其实是一种 **FOMO** 的驱动。

在 AI 和自媒体时代，一些容易吸引流量的技术关键词会被不断放大，让更多人觉得“必须跟上”。

但慢慢会发现，其实：

- 会安装某个 AI 工具
    
- 会跑某个模型

本身并不是一件特别重要的事情。

技术迭代太快了，你几乎不可能真正“追上”每一个新东西。

很多时候是：

- 你刚开始研究一个工具
    
- 还没完全搞明白
    
- 下一个工具又出来了
    

所以更重要的能力也许不是追逐每一个新技术，而是一些 **meta skills**：

- learning how to learn
    
- 和人交流获取信息
    
- 从生活和工作中发现真正的问题
    

技术会不断变化，但这些能力相对来说更长期。
